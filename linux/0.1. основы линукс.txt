1. основной интерпретатор - bash:
	предоставляет гораздо больше возможностей для взаимодействия с программами и процессами, позволяя быстрее получать информацию
	переменная PS1:
		может быть полезен при тестировании на проникновение для отслеживания действий:
			показывать IP-адрес цели
			отображать дату и время выполнения команд
			информировать о статусе последней команды (успех или ошибка)
		управляет внешним видом командной строки. позволяет персонализировать приглашение, добавляя такие элементы, как:
			имя пользователя (\u)
			имя хоста (\h)
			текущий каталог (\w)
			дата и время
			цвета и специальные символы
		список часто используемых символов для настройки приглашения:
			список		описание
			\u			Имя текущего пользователя
			\h			Короткое имя хоста
			\H			Полное имя хоста
			\w			Полный путь текущего рабочего каталога
			\W			Только имя текущего рабочего каталога
			\d			Дата (например, "Mon Feb 6")
			\D{%Y-%m-%d}		Дата в формате YYYY-MM-DD
			\t			Время в 24-часовом формате (HH:MM:SS)
			\T			Время в 12-часовом формате (HH:MM:SS)
			\@			Время в формате AM/PM
			\s			Имя оболочки
			\j			Количество задач, управляемых оболочкой
			\n			Новая строка
			\r			Возврат каретки
		инструменты для настройки приглашения:
			Bash Prompt Generator - веб-инструмент для создания персонализованного приглашения
			Powerline - расширение для красивого оформления приглашения с использованием значков и цветов
	команды для получения помощи:
		man <tool>: от слова manual предоставляет подробные руководства с детальными обьяснениями по использованию команд 
			используется с сочетанием другой команды (вместо <tool>), чтобы узнать о нем (например man man - мануал по мануалу)
			ответ содержит следующие главы:
				NAME - название команды и ее краткое описание
				SYNOPSIS - общий формат использования команды
				DESCRIPTION - подробное описание функций команды
				OPTIONS - список доступных параметров
		<tool> --help: позволяет быстро посмотреть доступные параметры без чтения полной документации
			альтернатива: <tool> -h доступно не для всех команд
		help <command>: не все команды существуют в man-страницах, но их можно найти через help
			вот несколько команд, которых нет в man-страницах: cd, echo, alias
		apropos <keyword>: команда для поиска по описаниям man-страниц по ключевому имени
			например apropos sudo - выдаст все команды, где встречается ключевое слово в (n) количестве
	команды для настройки терминала:
		clear: очищает окно от записей (или Ctrl + L)
		стрелки вверх и вниз: для пролистывания историй введенных команд
		<command1> && <command2>: позволяет выполнить несколько команд поочередно, если каждая последующая команда выполнена без ошибок
		<command1> ; <command2>: позволяет выполнить несколько команд поочередно, игнорируя результаты и ошибки прошлых команд
		<command1> | <command2>: позволяет переадресовать результаты первой команды на вторую
		<command> &: позволяет фоново работать программе. при окончании работы программы, результаты будут выведены
	сочетания клавиш:
		[CTRL] + C: для завершения работы запущенной в терминале программы
		[CTRL] + Z: для приостановки работы запущенной в терминале программы
		[CTRL] + D: для закрытия терминала
		[CTRL] + L: для очистки терминала
	команды для регулирования фоновыми программами:
		bg: команда для продолжения работы программы в фоновом режиме
			bg <jobs_id>: для продолжения работы определенной программы из списка jobs
		jobs: команда для выведения на экран всех фоновых процессов
			-l <PID>: отобразить задачу с указанным PID
		fg: для возврата фонового режима на передний план
			fg <jobs_id>: для возврата определенной программы из фонового режима на передний план

2. программы и их команды для управления пакетами программных обеспечений:
	dpkg - инструмент для установки, сборки, удаления и управления пакетами Debian. основным и более удобным интерфейсом для dpkg является aptitude
		-i <filename>: для установки пакета
	aptitude - высокоуровневый интерфейс для менеджера пакетов
	apt - альтернатива aptitude, представляет высокоуровневый интерфейс командной строки для управления пакетами
		list -i: для получения списка всех установленных пакетов
		update: для обновления информации о доступных новых пакетах
		upgrade: обновляет установленные пакеты до последних новых версий
		install <name>: для установки программного обеспечения
		remove <name>: удаляет пакет, не удаляя конфигурационные файлы
		purge <name>: удаляет пакет с конфигурационным файлом
		autoremove: удаляет неиспользуемые пакеты
		search <name>: ищет пакеты по имени или описанию в репозитории
		show <name>: показывает информацию о пакете

3. базовые функции команд:
	1. ключи - функции команд. позволяет гибко настроить программу на выполнение задачи. порядок расположения ключей неважен. бывают двух типов:
		-l: ключ, длиною в символ с одним дефисом
			при необходимости указании значения, указывается после ключа с пробелом (-l 80)
		--step: ключ с полным названием, указывается двумя дефис
			при необходимости указании значения, указывается после ключа со знаком равно (--step=90)
	2. сочетание ключей - имеется возможность совмещать ключи, вместо отдельных писанин (ls -l -a -h) -> (ls -lah)
	3. автодополнение TAB-ом - при двойном нажатии TAB система показывает все файлы в указанной папке, начинающиеся с букв
		пример: cd /dev/s [TAB x2] - выдаст значения, начинающиеся с буквы s; если значение одно, то команда будет дополнена автоматически (cd /dev/sh [TAB] -> cd/dev/shm)
