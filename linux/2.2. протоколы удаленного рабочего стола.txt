два наиболее распространенных протокола:
	RDP (Remote Desktop Protocol) - в основном используется в системах Windows
		позволяет подключаться и взаимодействовать с рабочим столом так, как если бы он находился прямо перед ним
	VNC (Virtual Network Computing) - в основном используется в средах Linux, несмотря на свою кроссплатформенность
		обеспечивает графический доступ и напрямую взаимодействовать с удаленной системой

XServer - часть системы Linux, которая отвечает за отображение графического интерфейса
	при наличии XServer у двух устройств Linux, RDP и VNC не нужны
	работает по 2м принципам:
		X Server - локальная часть, которая позволяет управлять устройством
		X Client - приложение, подключенное к серверу, которое отправляет ему запросы
	пример работы:
		ssh -X user@remote_host		# подключение к ssh-серверу с флагом -X
		флаг -X позволяет запустить программу на удаленном компьютере, но видеть его интерфейс на своем
	порты:
		находятся в диапазоне TCP/6001-6009
	недостатки:
		незашифрованная передача данных - можно обойти через туннелирование по SSH
			для этого на удаленном устройстве необходимо в файле /etc/ssh/sshd_config установить значение X11Forwarding yes

XDMCP - протокол для связи через UDP-порт 177. предоставляет управление удаленными сеансами
	небезопасный протокол, взламывается посредством man-in-the-middle

VNC (Virtual Network Computing) - система удаленного доступа к рабочему столу, основанная на протоколе RFB
	использует шифрование для защиты данных при передаче и требует аутентификации
	в основном используется в средах Linux, несмотря на свою кроссплатформенность
	есть 2 способа взаимодействия:
		1. обычный сервер предоставляет пользователю доступа к реальному экрану
		2. серверные программы позволяют пользователю управлять устройством через терминал
	XFCE4 - менеджер рабочего стола. необходим для установки, так как VNC и GNOME конкурируют друг с другом
		имя пакетов для установки: xfce4 xfce4-goodies
	TigerVNC - один из инструментов для удаленных подключений
		имя пакета: tigervnc-standalone-server
		установка и настройка TigerVNC-сервера:
			1. командой vncpasswd вызываем окно указания пароля для будущих подключений
			2. во время установки появляется скрытая папка в домашнем каталоге .vnc
			2.1. внутри папки необходимо создать файл startup и config, которые отвечают за создание сеанса и настройки соединения
			startup:
				#!/bin/bash
				unset SESSION_MANAGER
				unset DBUS_SESSION_BUS_ADDRESS
				/usr/bin/startxfce4
				[ -x /etc/vnc/xstartup ] && exec /etc/vnc/xstartup
				[ -r $HOME/.Xresources ] && xrdb $HOME/.Xresources
				x-window-manager &
				EOT
			config:
				geometry=1920x1080
				dpi=96
				EOT
			2.2. необходимо добавить файлу startup права
				chmod +x ~/.vnc/xstartup
			3. теперь можно запустить VNC-сервер. осуществляется командой vncserver
			команды:
				vncpasswd: указать пароль для будущих подключений
				vncserver: для запуска сервера
					-list: для отображения списка сеансов
		подключение к VNC-серверу:
			1. необходимо настроить ssh-подключение для защиты данных
				ssh -L 5901:127.0.0.1:5901 -N -f -l htb-student 10.129.14.130, где:
					5901: - задаем локальный порт
					127.0.0.1:5901 - адрес с портом на котором работает VNC-сервер
					-N: просит удаленное устройство	не выполнять команду
					-f: запускает ssh после ввода пароля
					-l htb-student: для указания имени пользователя
					10.129.14.130: ip-адрес удаленной машины, к которому устанавливается ssh-соединение
			2. осуществляется подключение командой xtightvncviewer localhost:5901
