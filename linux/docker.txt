docker - контейнер, который создается отдельно для каждого приложения, без личной ОС
	работает на ядре хоста Linux, но: в своей файловой системе, изолирован от других процессов
	суть: создается образ (Image) по принципу - готовая система + приложение (необ.). файл с разрешением dockerfile - скрипт, описывающий как собрать образ
	понятия:
		контейнер - запущенная система
		образ - файл, который запускает контейнер
		dockerfile - файл, который собирает образ
		volume - папка из хоста, проброшенная в контейнер
		network - изолированная сеть, где живут контейнеры
	примерный синтаксис: docker run -it ubuntu bash
	базовые ключи:
		docker: справочное окно
		-it: запустить интерактивно: через терминал
		ubuntu: использовать образ ubuntu:latest, скачивается с Docker Hub
		bash: команда внутри контейнера
		-v /<имя_файла>:/data: пробросить файлы в контейнер 
	ключи - управление контейнерами:
		docker build -t <dockerfile>: собрать контейнер
		docker run -it <name>: запустить контейнер
		docker ps: показывает запущенные контейнеры
		docker images: показывает доступные образы
		docker start <id>: запустить контейнер
		docker stop <id>: остановить контейнер
		docker exec -it <id> bash: зайти внутрь запущенного контейнера
		docker rm <id>: удалить контейнер
		docker rmi <image>: удалить образ
	
	dockerfile имеет следующую структуру:
		FROM ubuntu:22.04		# указывается система, которая будет служить ОС

		RUN apt update && apt install -y curl

		CMD ["bash"]			# указывается приложение, которое будет на основе (если не указывать приложение, docker запустится но сразу же закроется)
						# на данном случае, как приложение указан терминал bash
