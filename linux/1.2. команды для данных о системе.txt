команды для управления службами (демонами):
	systemctl <command> <daemon>: основной инструмент для управления системными службами и состоянием системы
		примерный синтаксис: systemctl status ssh
		команды для работы со службой:
			status <daemon>: показывает текущее состояние службы
			start <daemon>: запускает службу
			stop <daemon>: останавливает службу
			restart <daemon>: перезапускает службу
			reload <daemon>: перезапускает службу без полной остановки
			enable <daemon>: включает автозапуск службы
			disable <daemon>: отключает автозапуск службы
			is-enabled <daemon>: проверяет на автозапуск службу
			list-units --type=service: показывает список всех запущенных служб
			list-unit-files --type=service: показывает все службы, даже те, которые не запущены
		команды для работы с состоянием системы:
			hibernate: переход системы в режим гибернации
			suspend: переход системы в спящий режим
			poweroff: выключение системы
			reboot: перезагрузка системы
	ps <options>: для просмотра запущенных процессов
	journalctl <options> <daemon>: для просмотра лога о работе службы
	kill <options> <ID>: для завершения работы программы

команды для установления различного сведения о системе:
	whoami: отображает текущее имя пользователя
	id: возвращает идентификационные данные пользователей
	hostname: устанавливает или выводит на экран имя текущей хост-системы
	uname: выводит основную информацию о названии ОС и системном оборудовании
	ps: позволяет увидеть активные процессы
	who: показывает, кто вошел в систему
	env: выводит информацию о переменных окружения. позволяет изменять / просматривать / очищать
		переменные окружения - важная информация о системе и настройках пользователя
	lspci: для отображения информации об устройствах, подключенных к шине PCI: видеокарты, сетевые карты, звуковые карты контроллеры
