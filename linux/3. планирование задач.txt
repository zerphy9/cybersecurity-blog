команды для планирования задач:
	systemd - служба для запуска процессов и скриптов в определенное время / через определенный промежуток времени
		1. для работы необходимы файлы в корневой папке systemd, в файлах будут храниться скрипт таймера и службы соответственно
		2. необходимо обновить базу systemd для того чтобы программа видела, командой:
			sudo systemd daemon-reload
		3. необходимо запустить таймер и сервис, командами:
			sudo systemd start name.service
			sudo systemd enable name.timer
		файл со скриптом таймера:
			создается файл в адресе /etc/systemd/system/ с разрешением timer (name.timer), который содержит:
				[Unit]
				Description=My Timer

				[Timer]
				OnBootSec=3min
				OnUnitActiveSec=1hour

				[Install]
				WantedBy=timers.target
			в котором:
				в главе Unit - описание таймера
				в главе Timer - задать время запуска (через 3 минуты после запуска иситемы) и повтора (через час после срабатывания таймера)
				в главе Install - значение по умолчанию
		создание сервиса:
			создается файл в адресе /etc/systemd/system/ с разрешением service (name.service), в котором:
				[Unit]
				Description=My Service

				[Service]
				ExecStart=/full/path/to/my/script.sh

				[Install]
				WantedBy=multi-user.target
			в котором:
				в главе Unit - описание сервиса
				в главе Service - указывается полный путь к скрипту
				в главе Install - значение по умолчанию
	cron - инструмент для автоматического запуска процессов. использует файл с разрешением crontab
		пример файла с разрешением crontab:
			# System Update
			0 */6 * * * /path/to/update_software.sh

			# Execute Scripts
			0 0 1 * * /path/to/scripts/run_scripts.sh

			# Cleanup DB
			0 0 * * 0 /path/to/scripts/clean_database.sh

			# Backups
			0 0 * * 7 /path/to/scripts/backup.sh
		содержит в себе 5 значений и путь к файлу, каждое значение имеет в виду:
			Минуты (0-59)			Здесь указывается, в какую минуту должна быть выполнена задача.
			Часы (0-23)			Здесь указано, в какое время должна быть выполнена задача.
			Дни месяца (1-31)		Здесь указывается, в какой день месяца должна быть выполнена задача.
			Месяцы (1-12)			Здесь указано, в каком месяце должна быть выполнена задача.
			Дни недели (0-7) (ВС-СБ)	Здесь указывается, в какой день недели должна быть выполнена задача.
			* - неуказанное значение означает каждый раз при новом значении
			*/6 - указывает что каждые 6 значений показателя будет производиться скрипт
