управление разрешениями к файлам:
	icacls - утилита для детализации разрешений файлов NTFS, позволяет изменять разрешения к файлам
		значения наследований прав:
			CI - права наследуются папками внутри этой папки
			OI - права наследуются файлами внутри этой папки
			IO - права действуют только на обьекты внутри этой папки
			NP - запрет на дальнейшее наследование - обьекты наследуются, но не дальше 2 уровня
			I - унаследованные права от родительской папки
		значения разрешений пользователю:
			F - полный доступ: читать, писать, удалять, менять разрешения и владельца
			D - можно делать все, кроме изменения разрешения и владельца
			RX - читать и запускать исполняемые файлы
			R - только читать
			W - только записывать
		примерный синтаксис: icacls c:\windows
		ключи:
			icacls - выводит список разрешения к файлам в текущей папке
			<address> - выводит список разрешения к файлам в указанной папке
			<address> /grant <username>:<mode> - позволяет менять разрешение к файлу указанному пользователю
			<address> /remove <username> - отозвать присвоенные разрешения у пользователя
			<путь> /inheritance:r - удалить унаследованные от корня права
			<путь> /grant "имя_пользователя/группы:(права1)(права2)" - назначить пользователю новые права доступа

команды для работы с локальной сетью:
	New-SmbShare - для создания общего доступа к файлу/папке
		синтаксис: New-SmbShare -Name "будущее_название_в_сети" -Path "путь_к_файлу" -FullAccess "имя_пользователя/группы"
	Get-Smbshare - получить список расшаренных папок
	Revoke-SmbShareAccess - удалить пользователя/группу из доступа к общей файлу/папке
		синтаксис: Revoke-SmbShareAccess -Name "имя_файла_в_сети" -AccountName "имя_пользователя/группы" -Force
	Grant-SmbShareAccess - изменить Share-права пользователя
		примерный синтаксис: Grant-SmbShareAccess -Name "имя_файла_в_сети" -AccountName "имя_пользователя/группы" -AccessRight <новые_права> 
	

compmgmt.msc - встроенная утилита для администрирования локального или удаленного компьютера
	существуют следующие категории:
	1. system tools
		event viewer - просмотр системных журналов, ошибок, предупреждений
		shared folders - управление расшаренными папками, открытыми сессиями и файлами
		local users and groups - управление локальными пользователями и группами
		performance logs and alerts - мониторинг производительности
		device manager - управление драйверами и устройствами
	2. storage
		disk management - создание, удаление, форматирование разделов, назначение букв дисков
	3. services and applications
		services - запуск, остановка, настройка служб Windows
		WMI control - управление Windows Management Instrumentation
		Component services - расширенное администрирование COM+ и DCOM компонентов

виды сеансов Windows:
	интерактивный - инициируется пользователем, который проходит аутентификацию в локальной системе с помощью учетной записи
		инициируется: путем прямого входа в систему, 
			запроса доп. сеанса через команду runas
			подключением к удаленному рабочему столу
	неинтерактивный - инициируется системой, учетная запись без учетных данных
		используются для автоматического запуска служб и приложений без участия пользователя
		существуют 3 типа:
			локальная системная учетная запись (NT AUTHORITY\SYSTEM) - самая мощная учетная запись, обладает более широкими правами
			локальная учетная запись службы (NT AUTHORITY\LocalService) - привилегии на уровне пользователя, обладает ограниченными функциями
			учетная запись сетевой службы (NT AUTHORITY\NetworkService) - привилегии на уровне пользователя, устанавливает сеансы аутентификации для определенных сетевых служб 

идентификатор безопасности (SID):
	уникальный числовой идентификатор, который присваивается каждому обьекту безопасности: группе, пользователю, ПК, учетной записи службы
	именно по нему определяются права доступа к файлу (SID проверяется в списке разрешений ACL)
	пример SID:
		S-1-5-21-3623811015-3361044348-30300820-1013
		S - SID; 1 - версия SID; 5 - идентификатор авторитета; 21... - уникальный идентификатор домена или ПК; 1013 - RID (Relative ID) уникальный номер внутри домена или ПК
	SID встроенных пользователей и групп:
		S-1-5-18 - LocalSystem
		S-1-5-19 - LocalService
		S-1-5-20 - NetworkService
		S-1-5-544 - группа "Administrators"
		S-1-5-545 - группа "Users"
	команды:
		Get-LocalUser | Select Name, SID - для получения SID пользователя
		Get-LocalUser -Name "имя пользователя" | Select SID

понятия безопасности:
	SAM (Security Account Manager) - база данных, которая хранит:
		учетные записи; их SID; хэш паролей; принадлежность к группам; другие параметры авторизации
		используется службой lsass при входе в систему
		хранится по адресу: C:\Windows\System32\config\SAM
	ACL (Access Control List) - список правил, описывающих кто и что может делать с обьектом 
		каждый обьект в Windows имеет свою ACL
		состоит из записей ACE
	ACE (Access Control Entry) - одна запись в списке ACE, которая говорит:
		SID ... разрешено/запрещено выполнять действия X
		примеры действий: Read; Write; FullControl; Execute
		Get-Acl <filename> - для просмотра ACL файла
	UAC (User Account Control) - окно с подтверждением запуска от имени администратора

реестр Windows:
	иерархическая база данных, в которой система и установленные программы хранят:
		настройки; параметры пользователей; информацию об оборудовании; данные о приложениях
	основные ветки реестра:
		HKEY_LOCAL_MACHINE (HKLM) - настройки всей системы (железо, драйверы, службы)
		HKEY_CURRENT_USER (HKCU) - настройки текущего пользователя
		HKEY_CLASSES_ROOT (HKCR) - ассоциации типов файлов и COM-обьекты
		HKEY_USERS - настройки всех пользователей
		HKEY_CURRENT_CONFIG - профиль текущей конфигурации оборудования
	запустить: Win + R -> regedit
	типы значений:
		REG_SZ - строка
		REG_DWORD - число 32-бит
		REG_QWORD - число 64-бит
		REG_BINARY - байты
		REG_MULTI_SZ - список строк

Local Group Policy:
	утилита, с помощью которого можно управлять настройками и поведением системы без реестра
	можно настраивать:
		пароли и ограничения входа; доступ к панели управления; автозапуск программ; поведение Windows Update;
		файервол и безопасность; скрипты входа и выхода; контроль UAC; ограничения PowerShell
	запустить: Win + R -> gpedit.msc

Защитник Windows:
	встроенный антивирус, который бесплатно поставляется с операционными системами Windows
	команды:
		Get-MpComputerStatus - для проверки, какие параметры защиты включены
