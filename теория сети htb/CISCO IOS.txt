Cisco IOS - операционная система, которая управляет работой сетевых устройств Cisco, таких как маршрутизаторы и коммутаторы. содержит инструменты для управления, настройки и мониторинга сетей
	основные функции:
		маршрутизация - поддержка протоколов маршрутизации (например OSPF, BGP)
		коммутация - настройка VLAN, STP и других протоколов для работы с коммутаторами
		безопасность - защита сети через ACL, шифрование и аутентификацию
		услуги сети - DHCP, DNS, NAT и другие сервисы
		виртуализация - поддержка технологий, таких как VRF и VPLS
	способы управления:
		CLI (Command Line Interface) - основной метод управления. позволяет выполнять точные команды для настройки устройства
		GUI (Graphical User Interface) - графический интерфейс, для менее опытных пользователей
		удаленный доступ - через Telnet или SSH
	типы паролей:
		user password - используется для входа в систему. ограничивает доступ к устройству и его функциям
		enable password - позволяет зайти в режим "enable", в котором есть расширенные настройки и функции
		secret - пароль для защиты определенных функций и служб. часто используется для ограничения доступа к удаленным инструментам управления
		enable secret - дополнительно защищенный пароль для режима "enable". хранится в зашифрованном виде

Cisco Discovery Protocol (CDP) - протокол уровня 2, используемый устройствами Cisco для обмена информацией о соседних устройствах
	использование: обнаружение топологии сети; управление и устранение неполадок
	безопасность: рекомендуется отключить CDP, если он не требуется, чтобы предотвратить раскрытие конфиденциальной информации

Spanning Tree Protocol (STP) - протокол уровня 2, предотвращающий петли в сети
	преимущества: предотвращает зацикливание трафика; обеспечивает отказоустойчивость
	недостатки: блокировка портов снижает производительность; не поддерживает мультиплексирование

VLAN (Virtual Local Area Network):
	логическая группировка сетевых устройств, подключенных к определенным портам коммутатора
	позволяют создавать логические широковещательные домены, которые могут охватывать несколько физических сегментов сети
	преимущества использования:
		лучшая организация - устройства можно сегментировать по отделам, функциям и приложениям
		повышенная безопасность - сегментация сети предотвращает случаи несанкционированного доступа к трафику других VLAN
		упрощенное администрирование - не нужно учитывать физическое расположение устройств
		увеличение производительности - снижение широковещательного трафика освобождает полосу пропускания
	пример сегментации сети:
		отдел компании		VLAN ID		подсеть
		Серверы			VLAN 10		192.168.1.0/24
		Руководство		VLAN 20		192.168.2.0/24
		Финансы			VLAN 30		192.168.3.0/24
		HR				VLAN 40		192.168.4.0/24
		Маркетинг			VLAN 50		192.168.5.0/24
		Поддержка			VLAN 60		192.168.6.0/24
	диапазоны VLAN ID:
		поддерживает в диапазоне от 1 до 4094
		ID 0 и 4095 - зарезервированы
		ID 1 - VLAN по умолчанию (не рекомендуется изменять или удалять)
		ID 2-1001 - нормальный диапазон. настройки сохраняются в файле vlan.dat
		ID 1002-1005 - зарезервированы для Token Ring и FDDI
		ID 1006-4094 - расширенный диапазон. настройки не сохраняются
	типы назначения VLAN:
		статическое назначение - порты назначаются VLAN вручную через интерфейс коммутатора
			преимущества: простота настройки; высокая безопасность, так как порт всегда связан с определенной VLAN
			недостатки: необходимость ручной настройки для каждого коммутатора
		динамическое назначение - VLAN назначается автоматически на основе MAC-адреса или протокола. используется централизованная база данных VLAN
			преимущества: гибкость и мобильность устройств
			недостатки: повышенная административная нагрузка; низкая безопасность засчет возможности подделать MAC-адрес
	типы портов VLAN (2):
		Access Port - принадлежит только одной VLAN
			может передавать трафик только этой VLAN (или доп. VLAN для голосового трафика)
			все входящие пакеты считаются принадлежащими назначенной VLAN
		Trunk Port - может передавать трафик нескольких VLAN одновременно
			используется для соединения коммутаторов и маршрутизаторов
			используется одна из технологий передачи VLAN-информации (ISL или IEEE 802.1Q)
	технологии передачи VLAN-информации (2):
		ISL (Inter-Switch Link) - проприетарный протокол Cisco
			Инкапсулирует весь Ethernet-фрейм, добавляя 26-байтный заголовок и 4-байтный трейлер.
			Устаревший протокол, который сейчас редко используется.
		IEEE 802.1Q - стандартная технология, поддерживаемая большинством современных устройств
			Добавляет 4-байтный тег VLAN в существующий Ethernet-фрейм.
			Широко применяется благодаря совместимости с различными производителями оборудования.
	основы IEEE 802.1Q:
		был разработан для обеспечения взаимодействия VLAN между разными сетевыми устройствами
		Ethernet-фрейм - пакет данных, который содержит информацию о том, кто отправляет пакет, кому, с какой скоростью и прочее
		изменение формата Ethernet-фрейма IEEE 802.1Q - добавляются два поля:
			TPID (Tag Protocol Identifier) - 16-битное поле, со значением 0x8100 - для обозначения IEEE 802.1Q
			TCI (Tag Control Information) - 16-битное поле с тремя подполями:
				PCP (Priority Code Point): Указывает приоритет пакета (3 бита).
				DEI (Drop Eligible Indicator): Флаг, указывающий, может ли пакет быть отброшен при перегрузке (1 бит).
				VID (VLAN Identifier): Идентификатор VLAN (12 бит).
					количество VID занимает 12 бит, что значит 2^12 - 2 = 4094 VLAN можно создать
	безопасность VLAN и атаки:
		VLAN Hopping - атака позволяет трафику из одной VLAN быть видимым в другой без использования маршрутизатора
		Double-Tagging VLAN Hopping - более сложная атака, использующая двойное тегирование VLAN
	проблемы VLAN:
		ограничение в 4094 VLAN (12 бит для VID)
		недостатки STP (Spanning Tree Protocol): блокировка портов; отсутствие мультиплексирования
	VXLAN (Virtual eXtensible Local Area Network) - решение проблем VLAN
		это Layer 2 overlay поверх Layer 3 сети, предназначенный для масштабирования сетей в центрах обработки данных
		преимущества:
			Поддержка до 16 миллионов сегментов (24-битный VNI).
			Гибкость и масштабируемость для виртуализированных сред.
			Изоляция трафика между сегментами.
		принцип работы:
			Каждый VXLAN-сегмент идентифицируется уникальным VNI.
			Только виртуальные машины (VM) внутри одного VXLAN-сегмента могут взаимодействовать.
