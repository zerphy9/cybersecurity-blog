протокол - набор правил и стандартов о том, как программы или устройства должны обмениваться данными (как язык для людей)
	устанавливают: формат данных; порядок передачи; методы проверки целостности данных; способы обработки ошибок

ключевая терминология протоколов:
	протоколы безопасности:
		WEP (Wired Equivalment Privacy) - протокол для защиты беспроводных сетей. устарел, но встречается в старых устройствах
		WPA (Wi-Fi Protected Access) - более безопасная замена WEP. использует динамические ключи шифрования (TKIP или AES)
		TKIP (Temporal Key Integrity Protocol) - промежуточный протокол между WEP и современными стандартами. менее безопасен чем AES
		PEAP (Protected Extensible Authentication Protocol) - создает зашифрованный туннель для аутентификации пользователей. часто используется в корпоративных сетях
		EAP (Extensible Authentication Protocol) - фреймворк для поддержки разных методов аутентификации. поддерживает пароли, сертификаты, одноразовые пароли и др.
		LEAP (Lightweight Extensible Authentication Protocol) - проприетарный протокол Cisco для беспроводных сетей. основан на EAP, но менее безопасен
		IPsec (Internet Protocol Security) - протокол для создания защищенных соединений между устройствами. широко используется в VPN
		PGP (Pretty Good Privacy) - программа шифрования для защиты электронной почты, файлов и других данных
	протоколы передачи данных:
		FTP (File Transfer Protocol) - используется для передачи данных между компьютерами. небезопасен, так как данные отправляются в открытом виде
		HTTP (Hypertext Transfer Protocol) - протокол для передачи данных в интернете. его безопасная версия: HTTPS (использует SSL/TLS)
		SMTP (Simple Mail Transfer Protocol) - протокол для отправки и получения электронной почты
		NNTP (Network News Transfer Protocol) - используется для распространения и получения сообщений в группах новостей (newsgroups) через интернет	
		SMB (Server Message Block) - позволяет обмениваться файлами, принтерами и другими ресурсами в сети. используется в Windows для общих папок
		NFS (Network File System) - протокол для доступа к файлам через сеть. часто используется в Unix/Linux
	протоколы управления сетью:
		SNMP (Simple Network Management Protocol) - для мониторинга и управления сетевыми устройствами (прим. для проверки состояния маршрутизатора / коммутатора)
		CDP (Cisco Discovery Protocol) - позволяет обнаруживать Cisco-устройства в сети. полезен для диагностики
		STP (Spanning Tree Protocol) - предотвращает петли в сетях Ethernet. обеспечивает стабильную работу сети
		VTP (VLAN Trunking Protocol) - упрощает управление VLAN на нескольких коммутаторах. позволяет автоматически распространять конфигурацию VLAN
		TACACS (Terminal Access Controller Access-Control System) - протокол для централизованной аутентификации, авторизации и учёта (AAA) при доступе к сети
		MBSA (Microsoft Baseline Security Analyzer) - бесплатный инструмент от Microsoft для выявления потенциальных уязвимостей в системах Windows
	протоколы маршрутизации:
		RIP (Routing Information Protocol) - простой протокол маршрутизации. ограничен расстоянием (максимум 15 хопов)
		OSPF (Open Shortest Path First) - протокол маршрутизации для больших сетей. использует алгоритм Dijkstra для поиска кратчайшего пути
		IGRP (Interior Gateway Routing Protocol) - проприетарный протокол Cisco для маршрутизации внутри автономных систем. заменён более современным EIGRP
		EIGRP (Enhanced Interior Gateway Routing Protocol) - проприетарный протокол Cisco. быстрее и эффективнее RIP
	протоколы синхронизации и времени:
		NTP (Network Time Protocol) - используется для синхронизации времени на компьютерах в сети
	протоколы удаленного доступа:
		SSH (Secure Shell) - защищённый протокол для удалённого управления устройствами
		RSH (Remote Shell) - устаревший протокол для выполнения команд на удалённом компьютере. небезопасен, так как не использует шифрование.
		PPTP (Point-to-Point Tunneling Protocol) - протокол для создания VPN. устарел, так как менее безопасен, чем IPsec
	протоколы голосовой связи:
		SIP (Session Initiation Protocol) - используется для установления голосовых и видеосессий. основа VoIP
		VoIP (Voice over IP) - технология для передачи голоса через интернет. заменяет традиционные телефонные линии
	протоколы резервирования:
		HSRP (Hot Standby Router Protocol) - протокол Cisco для обеспечения отказоустойчивости маршрутизаторов. если один маршрутизатор выходит из строя, другой берёт на себя его функции
		VRRP (Virtual Router Redundancy Protocol) - стандартный протокол для резервирования маршрутизаторов. аналогичен HSRP, но работает на разных платформах
	протоколы для работы с VLAN:
		VLAN (Virtual Local Area Network) - логическое разделение физической сети на несколько независимых сетей
		VTP (VLAN Trunking Protocol) - упрощает управление VLAN на нескольких коммутаторах
	протоколы для создания защищённых соединений:
		VPN (Virtual Private Network) - технология для создания защищённого соединения через интернет
		GRE (Generic Routing Encapsulation) - протокол для инкапсуляции данных внутри туннеля VPN
		IKE (Internet Key Exchange) - протокол для установления безопасного соединения между двумя устройствами. используется в IPsec
	протоколы для работы с веб-серверами:
		ISAPI (Internet Server Application Programming Interface) - набор API для создания высокопроизводительных веб-расширений на веб-серверах, таких как Microsoft IIS
	протоколы для управления текстовыми данными:
		CRLF (Carriage Return Line Feed): комбинация символов \r\n, используемая для обозначения конца строки в текстовых файлах. часто применяется в HTTP-заголовках
	протоколы для управления системами:
		SMS (Systems Management Server) - решение для управления сетями, системами и мобильными устройствами
	протоколы для промышленных систем:
		SCADA (Supervisory Control and Data Acquisition) - промышленная система управления, используемая для мониторинга и контроля технологических процессов.
	другие важные термины:
		NAT (Network Address Translation) - позволяет нескольким устройствам использовать один публичный IP-адрес. используется в домашних роутерах
		URI/URL:
			URI — универсальный идентификатор ресурса
			URL — подмножество URI, которое указывает конкретный адрес (например, https://example.com)
		AJAX (Asynchronous JavaScript and XML) - технология для создания динамических веб-страниц. позволяет обновлять часть страницы без полной перезагрузки

основные типы протоколов - TCP и UDP для установления соединения:
	TCP (Transmission Control Protocol) - надежен в доставке данных, но медленная скорость работы из-за дополнительной нагрузки на установку и поддержание соединения
		используется для важных данных, таких как веб-страницы (HTTP/HTTPS), электронная почта (SMTP), файловые передачи (FTP)
		структура состоит из двух сегментов:
			1. заголовок (header) - содержит метаданные для управления соединением. основные поля:
				source port - порт отправителя
				destination port - порт получателя
				sequence number - номер последовательности для упорядочивания данных
				acknowledgment number - подтверждение получения данных
				control flags - флаги для управления соединением
				window size - сколько данных может принять получатель
				checksum - контрольная сумма для обнаружения ошибок в заголовке и полезной нагрузке
				urgent pointer - указывает на срочные данные в полезной нагрузке
			2. полезная нагрузка (payload) - фактические данные
	UDP (User Datagram Protocol) - отправляет пакеты без проверки быстро, но нет гарантии что данные дойдут до получателя
		структура состоит из:
			- портов получателя и отправителя
			- длины
			- контрольной суммы
	Протоколы для удаленного доступа
		Telnet (порт 23): Удаленный вход на сервер. Небезопасный, так как данные передаются в открытом виде.
		SSH (Secure Shell, порт 22): Безопасная версия Telnet, шифрует весь трафик.
		RDP (Remote Desktop Protocol, порт 3389): Используется для удаленного управления компьютерами.
	Протоколы для передачи файлов
		FTP (File Transfer Protocol, порты 20-21): Стандартный протокол для передачи файлов.
		TFTP (Trivial File Transfer Protocol, порт 69): Упрощенная версия FTP, используется для небольших задач.
		SCP (Secure Copy Protocol, порт 22): Безопасная передача файлов через SSH.
		SMB (Server Message Block, порт 445): Используется для обмена файлами в локальных сетях.
		NFS (Network File System, порты 111, 2049): Монтирование удаленных файловых систем.
	Протоколы для веб-трафика
		HTTP (Hyper Text Transfer Protocol, порт 80): Используется для передачи веб-страниц.
		HTTPS (Hyper Text Transfer Protocol Secure, порт 443): Защищенная версия HTTP с использованием SSL/TLS.
		SOAP (Simple Object Access Protocol, порты 80, 443): Протокол для веб-сервисов.
	Протоколы для работы с электронной почтой
		SMTP (Simple Mail Transfer Protocol, порт 25): Используется для отправки электронной почты.
		POP3 (Post Office Protocol, порт 110): Используется для получения почты.
		IMAP (Internet Message Access Protocol, порт 143): Более современный протокол для доступа к почте.
	Протоколы для управления сетью
		SNMP (Simple Network Management Protocol, порты 161-162): Управление сетевыми устройствами.
		DHCP (Dynamic Host Configuration Protocol, порты 67, 68): Автоматическая настройка IP-адресов.
		DNS (Domain Name System, порт 53): Преобразование доменных имен в IP-адреса.
		ICMP (Internet Control Message Protocol): Диагностика сетевых проблем (например, ping).
		IGMP (Internet Group Management Protocol): Мультикастинг (групповая рассылка).
	Протоколы для безопасности
		Kerberos (порт 88): Аутентификация и авторизация пользователей.
		RADIUS (Remote Authentication Dial-In User Service, порты 1812, 1813): Аутентификация для удаленного доступа.
		ISAKMP (Internet Security Association and Key Management Protocol, порт 500): Используется для создания защищенных VPN-соединений.
	Протоколы для баз данных
		Oracle DB Listener (порты 1521/1526): Получение запросов от клиентов Oracle.
		Microsoft SQL Server (ms-sql-s, порт 1433): Подключение клиентов к SQL Server.
		DB2 (порт 50000): Реляционная СУБД для корпоративных приложений.
	Протоколы для голосовой связи
		SIP (Session Initiation Protocol, порт 5060): Используется для VoIP-сеансов.
	Прочие протоколы
		NTP (Network Time Protocol, порт 123): Синхронизация времени между устройствами.
		LDAP (Lightweight Directory Access Protocol, порт 389): Доступ к каталогам (например, Active Directory).
		BOOTP (Bootstrap Protocol, порты 67, 68): Загрузка компьютеров через сеть.
		NNTP (Network News Transfer Protocol, порт 119): Доступ к новостным группам.
		X11 (порт 6000): Графический интерфейс для сетевых компьютеров.
	
	основные протоколы, работающие через UDP
		Протоколы для разрешения имен и адресов
			DNS (Domain Name System, порт 53): Преобразование доменных имен в IP-адреса. DNS часто использует UDP из-за его скорости.
			NetBIOS Name (netbios-ns, порт 137): Используется в Windows для преобразования NetBIOS-имен в IP-адреса.
		Протоколы для передачи файлов
			TFTP (Trivial File Transfer Protocol, порт 69): Упрощенный протокол передачи файлов, часто используется для загрузки конфигураций сетевых устройств.
		Протоколы для синхронизации времени
			NTP (Network Time Protocol, порт 123): Синхронизация времени между устройствами в сети.
		Протоколы для управления сетью
			SNMP (Simple Network Management Protocol, порт 161): Мониторинг и управление сетевыми устройствами удаленно.
			RIP (Routing Information Protocol, порт 520): Обмен маршрутной информацией между маршрутизаторами.
		Протоколы для удаленного доступа
			Telnet (порт 23): Текстовый протокол для удаленного доступа к устройствам.
			VNC (Virtual Network Computing, порт 5900): Графический протокол для удаленного управления компьютерами.
			X11 (X Window System, порты 6000-6063): Графический интерфейс для Unix-подобных систем.
		Протоколы для баз данных
			MySQL (порт 3306): Открытая система управления базами данных.
			PostgreSQL (PGSQL, порт 5432): Объектно-реляционная СУБД.
			Microsoft SQL Server (ms-sql-m, порт 1434): Используется для службы Microsoft SQL Server Browser.
		Протоколы для мультимедиа и общения
			UPnP (Universal Plug and Play, порт 1900): Автоматическое обнаружение устройств в сети.
			IRC (Internet Relay Chat, порт 194): Протокол для текстового общения в реальном времени.
			OpenPGP (порт 11371): Шифрование и подписание данных.
		Протоколы для безопасности
			IPsec (Internet Protocol Security, порт 500): Создание защищенных туннелей (например, в VPN).
			IKE (Internet Key Exchange, порт 500/11371): Протокол для обмена ключами шифрования.
		Прочие протоколы
			BOOTP (Bootstrap Protocol, порт 68): Загрузка устройств через сеть.
			DHCP (Dynamic Host Configuration Protocol, порт 67): Автоматическая настройка IP-адресов.
			Syslog (порт 514): Сбор и хранение логов на компьютерах.
			XDMCP (X Display Manager Control Protocol, порт 177): Удаленный вход в системы с графическим интерфейсом X11.

Internet Control Message Protocol (ICMP) - протокол, который используется диагностики сетей:
	служит для отчетов об ошибках; статусных сообщений;
	работает на 3м уровне OSI
	основные функции ICMP:
		1. проверка доступности устройства
			Echo Request - Echo Reply: устройство отправляет запрос другому устройству, чтобы проверить доступно ли оно. 
			команда ping работает на этой основе
		2. сообщения об ошибках
			если пакет не может быть доставлен целевому устройству, он отправляет взаимный ответ с ошибкой
			Destination Unreachable (недостижимая цель): Целевой адрес недоступен.
			Time Exceeded (время истекло): Пакет превысил лимит времени (TTL = 0).
			Parameter Problem (проблема с параметрами): Некорректный заголовок пакета.
		3. перенаправление трафика
			Redirect (перенаправление): целевое устройство (маршрутизатор) сообщает устройству, что оно должно использовать другой маршрут для отправки пакетов
		4. контроль перегрузок
			Source Quench (источник замедления):целевое устройство (маршрутизатор) сообщает устройству, что оно доставляет пакеты очень быстро и не успевает их обрабатывать
	версии ICMP:
		ICMPv4 - разработан для работы с IPv4; используется для диагностики сетей, отчетов об ошибках и других задач
		ICMPv6 - разработан для работы с IPv6; включает дополнительные функции, такие как поддержка Neighbor Discovery Protocol (NDP), который используется для обнаружения соседей в сети
			ICMPv6 также используется для Stateless Address Autoconfiguration (SLAAC), которая позволяет устройствам автоматически настраивать свои IPv6-адреса
	поле TTL (Time-To-Live):
		поле в заголовке IP-пакета, которое ограничивает время жизни пакета в сети
			каждый раз, когда пакет передается другому маршрутизатору, значение TTL уменьшается на 1
			когда TTL равняется 0, пакет удаляется, а отправителю возвращается ошибка Time Exceeded
		необходим для ограничения зацикливания - предотвращает бесконечное перемещение пакетов по сети из-за ошибок маршрутизации
		позволяет узнать расстояние до цели: если TTL=122, а исходное было 128 -> пакет прошел 6 маршрутизаторов
		позволяет узнать ОС отправителя (не точно):
			windows - по умолчанию 128
			linux/macOS - по умолчанию 64
			solaris - по умолчанию 255

Voice over Internet Protocol (VoIP) - протокол, позволяющий передавать голосовые и мультимедийные данные через интернет
	преимущества над телефонными линиями:
		- звонки дешевле или бесплатны
		- возможность интеграции с другими сервисами
		- легко добавлять новых пользователей и настраивать новые функции
	протоколы, используемые в VoIP:
		SIP (Session Initiation Protocol) - для инициации, поддержания, изменения и завершения сеансов звонков в реальном времени
			порты: используется TCP/5060 (незашифрованный) и TCP/5061 (зашифрованный)
			методы:
				INVITE: Инициирует сеанс (например, звонок).
				ACK: Подтверждает получение запроса INVITE.
				BYE: Завершает сеанс.
				CANCEL: Отменяет ожидающий запрос INVITE.
				REGISTER: Регистрирует устройство (пользовательский агент) на SIP-сервере.
				OPTIONS: Запрашивает информацию о возможностях сервера или устройства.
		H.323 - набор стандартов для мультимедийной связи через пакетные сети. менее популярен чем SIP, но используется в некоторых корпоративных системах
			порты: часто используется TCP/1720
		RTP (Real-time Transport Protocol) - передача аудио и видео данных в реальном времени
			порты: работает поверх UDP для обеспечения скорости передачи данных
	безопасность:
		атаки на VoIP-системы:
			Перебор учетных записей: Злоумышленники могут использовать запросы SIP OPTIONS для проверки существования пользователей.
				позволяет получить: поддерживаемые кодеки; типы мультимедиа; доступность пользователя
			Сканы портов: Поиск открытых портов (например, 5060, 5061) может выявить активные VoIP-серверы.
			Атаки типа "Denial of Service" (DoS): Перегрузка VoIP-серверов может привести к сбоям в работе системы.
			Подслушивание: Перехват RTP-трафика может позволить злоумышленникам прослушивать разговоры.
		защита VoIP-системы:
			Шифрование: Использование защищенных протоколов (например, SRTP для RTP-трафика).
			Фильтрация трафика: Настройка брандмауэров для ограничения доступа к VoIP-портам.
			Авторизация и аутентификация: Использование сложных паролей и двухфакторной аутентификации.
			Обновление ПО: Регулярное обновление прошивки и программного обеспечения для устранения уязвимостей.	

аутентификационные протоколы:
	обеспечивают безопасную и стандартизованную проверку подлинности пользователей, устройств, или других сущностей в сети (прим. авторизация в вк)
	без них невозможно надежно идентифицировать участников сети, что делает сеть уязвимой для несанкционированного доступа и атак
	основные протоколы:
		Kerberos - протокол аутентификации, использующий центр распределения ключей (KDC) и тикеты для доменных сред
			преимущества:
				высокая безопасность благодаря шифрованию; широко используется в корпоративных сетях
			недостатки:
				требует настройки KDC
		SRP (Secure Remote Password) - парольный протокол аутентификации, защищенный криптографией
			преимущества:
				устойчив к перехвату и MITM-атакам; не требует передачи пароля в открытом виде
		SSL/TLS - криптографические протоколы для безопасной аутентификации через сеть
			преимущества:
				широко поддерживаются; используются для защиты HTTPS-трафика
			недостатки:
				уязвимости старых версий (прим. SSL 3.0)
		OAuth - открытый стандарт аутентификации, позволяющий пользователям предоставлять доступ к ресурсам без передачи паролей
			применяется в API-авторизации (прим. Google, Facebook)
		OpenID - децентрализованный протокол аутентификации для использования единой учетной записи
			позволяет входить на различные сайты с одной учетной записью
		SAML (Security Assertion Markup Language) - XML-стандарт для обмена данными аутентификации и авторизации
			применяется как корпоративная аутентификация (SSO)
		2FA/MFA - методы аутентификации, использующие два и более фактора
			- что то, что пользователь знает (пароль)
			- что то, что пользователь имеет (телефон, сим-карта)
			- что то, чем является пользователь (биометрия)
			премущество - высокая безопасность
		PAP/CHAP:
			PAP (Password Authentication Protocol) - простой протокой, передающий пароль в открытом виде. небезопасен
			CHAP (Challenge Handshake Authentication Protocol) - использует тройное рукопожатие для проверки подлинности
		EAP (Extensible Authentication Protocol) - фреймворк (шаблон) для поддержки разных методов аутентификации
			применяется для беспроводных сетей (Wi-Fi), VPN
		LEAP (Lightweight EAP) - протокол аутентификации для беспроводных сетей, разработанный Cisco
			недостатки:
				уязвим к атакам по словарю; использует слабое шифрование (RC4)
		PEAP (Protected EAP) - безопасный протокол, использующий TLS для шифрования
			преимущества:
				высокая безопасность; поддержка сертификатов и многофакторной аутентификации
			применяется в корпоративных сетях (Wi-Fi, VPN)
		SSH - протокол для безопасного удаленного доступа и выполнения команд
			преимущества:
				шифрование данных; поддержка PKI и сертификатов
		HTTPS - безопасная версия HTTP, использующая SSL/TLS
			применяется для защиты веб-трафика
